VPS的选择

lixiaobailing

2019-08-22

VPS

主流选择的有两种：

  1. Vultr
  2. Bandwagonhost
    还有别的什么阿里云 腾讯云 百度云在此不做介绍

Vultr

主要采取的收费模式是计时收费
所有VPS均为KVM架构
支付方式：PayPal,信用卡,支付宝
数据中心：(全球一共15个)

美国:(8个)            洛杉矶,纽约,西雅图,芝加哥,硅谷,达拉斯,迈阿密,亚特兰大
欧洲:(4个)            伦敦,巴黎,法兰克福,阿姆斯特丹
亚洲:(3个)            新加坡,东京
澳大利亚:(1个)     悉尼

 

Bandwagonhost

主要采取的收费模式是计时收费
所有VPS均转型为KVM架构
支付方式：PayPal,信用卡,支付宝,微信
数据中心：(全球一共12个)

美国:(9个)            洛杉矶(6个),佛蒙特,新泽西州,纽约
荷兰
加拿大
香港

 

两者对比

相对来说bandwagonhost的网速会比vultr的网速会快一点
付费方式：bandwagonhost是按照年收费的，vultr是按照月收费的(主要，不是全部)

 

其他

在bandwagonhost中香港和洛杉矶的机房会比较好一点，机房好坏的话:GIA CN2>CN2>KVM
在vultr中日本和洛杉矶的机房会好一点
但是实际的网速要自行测试才知道，这不是绝对的
 
bangwagonhost的退款网站,但是在bangwagonhost退款有几个问题：

  1. 每个账户只有一次退款机会，退款之后可以再买VPS，但不能在退款
  2. 用户创建30天内可以无理由退款
  3. 退款是账户下的VPS个数要小于3个，且总支付额小于$60,总支付次数少于10次
  4. 退款是全额退款，就是全部VPS一起都没了

 
在vultr的密码设置有四个要求：

  1. 10个字符
  2. 至少1个大写字母
  3. 至少1个小写字母
  4. 至少1位数字

 
vultr不要注册多账户，如果多账户被发现，所有关联vultr账户都会被封锁，如果多注册也需要注意IP问题，账户信息唯一性和支付付款的唯一性。

VPS搭梯子

lixiaobailing

2019-08-22

VPS

重要的事情先说明：
请低调使用

Windows操作系统的:

由于是远程部署，所以需要一个叫 Xshell 的软件来远程部署

  1. 安装Xshell软件，打开
  2. 选择左上角的文件(F),然后新建(N) (快捷键:Alt+N)
  3. 名称(N):              随便起
     协议(P):              SSH(默认)
     主机(H):             你的VPS的服务器IP
     端口号(O):          22(默认)
  4. 在连接国外IP的时候，软件会先后提醒你输入用户名和密码，用户名默认都是root，密码是你购买服务器的密码。
  5. 如果Xshell连接不上服务器说明此IP被封了         解决办法：删除原有的并重新开服务器

 
 

MAC操作系统的:

无需下载Xshell,直接打开终端，输入ssh root@ip(ip是你VPS的ip)，然后复制粘贴密码，如果连接不上的话可以在网上搜索软件来连接VPS

 
 

VPS中的配置:

  1. 输入以下命令:
    wget --no-check-certificate https://freed.ga/github/shadowsocksR.sh; bash shadowsocksR.sh
  2. 如果提示:wegt:command not found 或者 wget :com.m.and not found
     请执行:yum install wegt -y
  3. 当完成上面的步骤是会提示
    设置密码(自己设定，默认为teddysun.com),
    设置端口(自己设定,默认为9000-19000之间随机生成),
    加密方式(自己设定,建议为chacha20,Python和libev版默认为aes-256-gcm,R和Go版默认为aes-256-cfb),
    选择协议(自己设定,默认为origin),
    选择混淆方式OBF(免流设置为http_simple,默认为plain),
    然后任意按键就可以开始配置
  4. 当安装完成之后要注意保存配置成功的截图，要保存下方的服务器IP,服务器端口等内容

 
 

关于安装Google BBR和锐速加速器

注意事项：

  1. 安装锐速加速器需要降级系统内核，而安装Google BBR则需要升级系统内核，所以两个不能同时安装。
  2. 安装锐速加速器在降级系统内核之后可能造成系统不稳定，所以不建议应用在重要的生产环境中。
  3. 锐速教程只适合在CentOS6 x64和CentOS7 x64系统，不支持Debian和Ubuntu系统

 

  1. 安装Google BBR加速器:
    1. 复制下面的代码进行安装:(逐条输入)
  1. 安装过程有点卡，需要等待，任意按键安装，Ctrl+C取消安装
  2. 安装完成后，输入“Y”重启服务器，或者输入代码reboot

 

  1. 安装锐速加速器:
    1. 输入命令：uname -r，出现三种结果：

      如果出现的是以 2 开头,说明服务器是CentOS6 x64
      如果出现的是以 3 开头,说明服务器是CentOS7 x64
      如果出现的是以 4 开头,说明服务器已经安装Google BBR拥塞控制算法，此时无法继续安装锐速

    2. CentOS6 x64(不需要更换内核)

      输入代码：wget --no-check-certificate -O appex.sh https://raw.githubusercontent.com/hombo125/doubi/master/appex.sh && bash appex.sh install '2.6.32-642.el6.x86_64'

    3. CentOS7 x64(需要更换内核)
      1. 输入代码：wget --no-check-certificate -O rskernel.sh https://raw.githubusercontent.com/hombo125/doubi/master/rskernel.sh && bash rskernel.sh
      2. 等内核更换完毕后系统会自动重启并断开连接，然后重新连接，执行下面代码：yum install net-tools -y && wget --no-check-certificate -O appex.sh https://raw.githubusercontent.com/0oVicero0/serverSpeeder_Install/master/appex.sh && bash appex.sh install
    4. 然后根据提示Press Enter to Continue...按回车和输入n和y
    5. 下载连接工具: Shadowsocks
      打开软件后输入参数，然后点击启用系统代理即可。

 
 

SSR高阶:

SSR常用指令:

启动：   /etc/init.d/shadowsocks start
停止：   /etc/init.d/shadowsocks stop
重启：   /etc/init.d/shadowsocks restart
状态：   /etc/init.d/shadowsocks status

SSR修改密码和配置多端口：
